Installation

Get started with web-tui in your project. Choose the installation method that fits your setup.

Package Installation

Install web-tui using your preferred package manager:

npm install web-tui
bun add web-tui
yarn add web-tui
pnpm add web-tui

Usage Options

web-tui can be used in multiple ways depending on your project setup:

Option 1: Pure CSS (No Framework)

Import the CSS file directly in your HTML:

<link rel="stylesheet" href="node_modules/web-tui/src/css/web-tui.css">

Or import in your CSS/JavaScript:

/* In CSS */ @import 'web-tui/css'; /* Or in JavaScript (with bundler) */ import 'web-tui/css';

Then use the classes in your HTML:

<div class="tui-window"> <div class="tui-window-header"> <div class="tui-window-controls"> <button class="tui-window-control tui-window-control-close"></button> <button class="tui-window-control tui-window-control-minimize"></button> <button class="tui-window-control tui-window-control-maximize"></button> </div> <span class="tui-window-title">Terminal</span> </div> <div class="tui-window-body"> <p class="tui-text">Hello, World!</p> </div> </div>
Option 2: Tailwind Plugin

Add the plugin to your tailwind.config.js:

module.exports = { plugins: [ require('web-tui/plugin') ] }

This gives you:

  • All .tui-* component classes
  • terminal-* color utilities (e.g., text-terminal-cyan, bg-terminal-panel)
  • CSS custom properties for theming
  • Extended theme with terminal-specific values

Use terminal colors with Tailwind:

<div class="bg-terminal-panel text-terminal-cyan border-terminal-grid"> <span class="text-terminal-success">Success!</span> <span class="text-terminal-brightRed">Error</span> </div>
Option 3: React Components

Import and use React components directly:

import { TerminalWindow, Box, Text, Button, TerminalProvider } from 'web-tui/react'; function App() { return ( <TerminalProvider> <TerminalWindow title="My App"> <Box bordered title="Welcome"> <Text variant="success">Hello, World!</Text> <Button variant="primary">Click Me</Button> </Box> </TerminalWindow> </TerminalProvider> ); }

Note: React is an optional peer dependency. The library works without React if you only use CSS classes.

Option 4: Vanilla JS Utilities

For enhanced interactive components, import the minimal JS utilities:

import { initTabs, initDialog, initMenu, initTheme } from 'web-tui/js'; // Initialize tabs with JS enhancement const tabs = initTabs('.my-tabs'); tabs.activate('tab-2'); // Initialize dialog const dialog = initDialog('.my-dialog'); document.querySelector('#open-btn').onclick = () => dialog.open(); // Initialize theme switcher const theme = initTheme({ defaultTheme: 'dark' }); theme.setTheme('matrix-green');

Or use auto-initialization:

<body data-tui-auto-init> <div class="tui-tabs" data-auto-init> <!-- tabs content --> </div> </body> <script type="module"> import { autoInit } from 'web-tui/js'; autoInit(); </script>

Note: Most components work with CSS-only. JS utilities are optional and provide enhanced behavior like keyboard navigation and programmatic control.

CDN Usage

For quick prototyping, you can use a CDN (once published to npm):

<!-- CSS only --> <link rel="stylesheet" href="https://unpkg.com/web-tui/src/css/web-tui.css"> <!-- With JS utilities --> <script type="module"> import { initTheme } from 'https://unpkg.com/web-tui/src/css/web-tui.js'; initTheme(); </script>
